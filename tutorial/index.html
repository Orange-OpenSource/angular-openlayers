angular-openlayers tutorial

Foreword

angular-openlayers is a set of Angular directives for OpenLayers. angular-openlayers is not endorsed by OpenLayers nor Angular, but its developers do their best to keep it up-to-date with both projects.

The following document demonstrates angular-openlayers main features and is the entry point for developers willing to use it.

Using angular-openlayers

angular-openlayers requires both Angular and OpenLayers. Installing those may be achieved in different ways, depending on what suits you best. In the end, your html file should be similar to this.
<html> <head> <link rel="stylesheet" href="path/to/ol.css"> <link rel="stylesheet" href="path/to/angular-openlayers.css"> <script src="path/to/angular.js"></script> <script src="path/to/angular-openlayers.js"></script> <script src="path/to/ol.js"></script> <script>angular.module('example',['angular-openlayers']);</script> </head> <body ng-app="example"> [...] </body> </html>
Notice the line where we tell angular that the module 'example' depends on angular-openlayers.

In angular-openlayers, the <aol-map> is the base directive. It creates a map instance which is further configured by children directives. In this simple example, we make use of the <aol-layer-tile> and the <aol-source-osm> directive to tell the map to fetch the tiles from the OpenStreetMap project servers.

Specifying the zoom level and map center

By default, the map is centered on the spherical coordinates (latitude: 0, longitude: 0) and is zoomed-out. It is possible to change this behavior using the <aol-view> directive. Here, we center the map at the coordinates (latitude: 45.76, longitude: 4.84); i.e Lyon; and set up the zoom level so that the view encompass the whole city.

When specifying coordinates, one must use the <aol-coordinate> directive. This directive has 3 attributes: aol-x, aol-y and aol-projection.

  • aol-xand aol-y are the coordinates in the coordinate system. The corresponding projection; i.e. coordinate system to the projected coordinate system; is specified using the aol-projection directive.
  • aol-projection specifies which projection is to be used to translate the aol-x and aol-y coordinates into the projected (2D) view. When aol-projection is omitted, the coordinates are assumed to be expressed in their projected form i.e. EPSG:3857.

Using multiple tiles providers

In the previous example, we've seen how to fetch tiles from the OpenStreetMap project servers. Let's discover how to fetch tiles from other providers.

angular-openlayers is built upon OpenLayers which provides handlers; i.e. "sources"; for a vast majority of existing tile servers. Here, we use the source for the BingMaps tile servers. This source is accessible in angular-openlayers with the <aol-source-bingmaps> directive. Similarly, the other sources available in OpenLayers are available using the <aol-source-*> directives. Showing features on the map

This example shows how to add a "feature" to the map at a given location, here at (latitude: 45.76, longiture: 4.84). Please notice the small light-blue circle on the bridged labeled "Pont Wilson".

In this example, we add a vector layer (features being vectors) using the <aol-layer-vector> directive. we add a directive to the map after the tile layer directive (the order of the layers directives dictates their display order). Similarly to tile layers, we also add a source to the layer using the <aol-source-vector>. Finally, we add features to the source using the <aol-feature> directive.

The <aol-feature> directive requires a geometry to be displayed. The geometry is created using one of the <aol-geometry-*> directives, here <aol-geometry-point>. Geometries are described by their coordinate(s). In the previous example, the point geometry is described by a single coordinate, i.e. the location (latitude: 45.76, longitude: 4.84).

When a geometry requires more than one coordinate, the coordinates are given using the <aol-collection-coordinate> directive. In the following example, we define a feature showing a linestring geometry made of 2 segments (3 coordinates). To that effect, we use the <aol-geometry-linestring> directive.

Styling features

By default, features are represented using blue strokes. The <aol-style-*> directives allow specifying custom styles.

In this example, we have depicted the point geometry feature as a circle filled with a light green color a and black stroke. We also styled the segments so that they show up magenta and slightly thicker than with the default settings.

Moving features

The very reason why we've decided to develop angular-openlayers is to be able to display maps that are dynamically updated on model changes. One such requirement is to move features on the map when its coordinates are updated in the model. The following example shows how to do so.

Updating features styles

Similarly to coordinates, style directives reflect changes made to their model, e.g. change the color attribute of an aol-fill directive and the styled elements will have an updated color. The following example demonstrate this behaviour.

[angular controller] angular.module('example').controller('updatingFeatureStyleController', function($interval){ var _this = this; _this.featureColor = '#FF0000'; }); [html file]
